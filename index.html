What is The Von Neumann Architecture?

The Von Neumann architecture is a computer architecture that was first published by John Von Neumann in 1945.

Before 1945, a series of switches had to be manually turned, or read from a paper stream to run programs.

The Von Neumann architecture is based on the stored-program computer concept, where instruction data and program data are stored in the same memory. This design is still used in most computers produced today.

How it Works

The Fetch-Decode-Execute Cycle

  1. Fetch

    2. Instructions and data are fetched from the computer memory.

  2. Decode

    3. The data and instructions will then be translated to instructions understandable by the CPU

  3. Execute

    4. The CPU will then execute the instructions in a logical sequence.
    Once complete, the process will start again for the next program

Inside the CPU

Control Unit ALU

Arithmetic & Logic Unit PC

Control Unit

The Control Unit is in charge of controlling and coordinating the operations of the computer by controlling when and which instructions are to be executed.

Like a manager at a store that "controls" and "coordinates" the workflow.

ALU

The ALU is responsible for all the mathematics and calculations that need to be performed by the CPU. It can however only calculate:

  1. Addition
  2. Subtraction
  3. Multiplication
  4. Division
  5. Logical Comparisons. eg; Is 9 > 5

The ALU also includes Boolean operators. Boolean operators are the words "AND", "OR" and "NOT". They are used to make searches more precise. eg; Search for "Cats OR Dogs". This search will provide results for info related to either 'only cats', 'only dogs' or both. Click Here to learn more about Boolean Operators

Program Counter

The Program Counter is used to store the address in memory of the next instruction to be executed. In other words, it contains the memory location address of the instruction to be fetched from the memory.

General Purpose Register

The GPR can be thought of as the notepad of the CPU. This is used to hold data when performing calculations. If the ALU needs to temporarily store data as part of its calculations, the GPR is where it is stored.

Other Registers Used

MAR (Memory Address Register)

Contains the address of the current instruction to be fetched.

MDR (Memory Data Register)

Contains the instruction / data after it is fetched from memory.

PC (Program Counter)

Contains the address of the next instruction to be fetched.

CIR (Current Instruction Register)

Contains the instruction / data to be decoded.
